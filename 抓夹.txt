### 为什么 `ixx="0.0"` 会毁掉整个仿真？

在 Gazebo 这样的物理引擎中，计算物体旋转加速度使用的是牛顿-欧拉方程的角动量形式，即 $\alpha = \tau / I$（角加速度 = 扭矩 / 转动惯量）。

看你代码中 `link6_1_1` 的参数：
```xml
  <inertial>
    ...
    <inertia ixx="0.0" iyy="1e-06" izz="0.0" ixy="0.0" iyz="0.0" ixz="-0.0"/>
  </inertial>
```
你的转动惯量 $I_{xx}$ 和 $I_{zz}$ 竟然是 `0.0`！
在数学上，当引擎试图对这个关节施加哪怕一点点力矩时，都会发生**除以零（Division by Zero）**的灾难。这会导致物理计算结果变成无穷大（`NaN`），底层控制器直接崩溃死锁。这就是为什么你觉得“第六节结构有问题”，且 MoveIt 的轨迹规划一旦下发到底层就会报错的原因。同时，由于底层物理计算崩溃，挂在它周围的相机插件也会受到波及而停止工作。

### 🚀 终极抢救：赋予第六关节和抓夹正确的物理属性

请打开你的 `dummy-ros2_description/urdf/dummy-ros2.xacro` 文件，做以下**两次**小修改：

#### 1. 修复第六关节 (`link6_1_1`)
找到大约第 122 行的 `<link name="link6_1_1">`，把它里面的 `<inertial>` 标签替换成这样微小但非零的值：
```xml
  <inertial>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <mass value="0.05"/>
    <inertia ixx="0.00001" iyy="0.00001" izz="0.00001" ixy="0.0" iyz="0.0" ixz="0.0"/>
  </inertial>
```

#### 2. 为抓夹加上物理属性
为了防止我们刚加的虚拟抓夹因为同样的原因（没写惯性参数）导致控制器再次崩溃，请滑到文件最下方。
给 `gripper_base_link`、`left_finger`、`right_finger` 这三个 Link 各加上一段相同的、基础的惯性参数：
```xml
  <link name="gripper_base_link">
    <inertial>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <mass value="0.05"/>
      <inertia ixx="0.00001" iyy="0.00001" izz="0.00001" ixy="0.0" iyz="0.0" ixz="0.0"/>
    </inertial>
    <visual> ... </visual>
    <collision> ... </collision>
  </link>
```
*(对 `left_finger` 和 `right_finger` 也进行同样的添加)*。
