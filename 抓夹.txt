要实现当你输入 `(0, 0, 90°, 0, 0, 0)` 时，机械臂呈现图中“7”字型的姿态（即前两个关节垂直，第三个关节转动 90 度），你需要在 URDF 中对 **Joint3** 的物理初始位置（安装位姿）进行调整。

目前的 URDF 中，Joint3 的 `origin` 是没有旋转的（`rpy="0 0 0"`），这意味着当控制指令为 0 时，Link 3 是相对于 Link 2 保持直连状态。

### 修改方法

你需要修改 `Joint3` 的 `origin` 属性。根据你的描述，有两种逻辑方案，推荐使用**方案一**：

#### 方案一：修改物理安装角度（最直接）
我们将 Joint3 的物理初始位置旋转 90 度。这样，在 ROS2 控制器看来，现在的“弯曲”就是它的 0 位。

找到 URDF 中的 `Joint3` 部分，将 `rpy` 修改如下：

```xml
<joint name="Joint3" type="revolute">
  <origin xyz="0.03665 0.0 0.146" rpy="0 1.5708 0"/> 
  <parent link="link2_1_1"/>
  <child link="link3_1_1"/>
  <axis xyz="-1.0 0.0 0.0"/> <limit
    lower="-3.14"
    upper="3.14"
    effort="150"
    velocity="3.15" />
</joint>
```

#### 方案二：如果实物 0 位是垂直的，只需在控制时偏移
如果你希望软件里发送 `1.57` (90°) 时它才变成“7”字型，而发送 `0` 时它是垂直的，那么你**不需要修改 URDF**。你只需要在发送控制指令（例如通过命令行或代码）时，给 Joint3 赋值 `1.57` 即可：

* **命令行测试命令：**
    ```bash
    ros2 topic pub /forward_position_controller/commands std_msgs/msg/Float64MultiArray "{data: [0.0, 0.0, 1.5708, 0.0, 0.0, 0.0]}"
    ```

### 关键点排查
1.  **弧度制**：ROS2 Control 接收的是**弧度**（Radian）。$90^\circ = 1.570796$ 弧度。如果你输入数字 `90`，机械臂会疯狂旋转。
2.  **坐标系对齐**：修改 `rpy` 后，如果发现 Link3 的旋转中心偏移了，你可能需要同时微调 `link3_1_1` 标签下的 `<visual>` 和 `<collision>` 的 `origin`。
3.  **方向问题**：如果修改 `rpy="0 1.5708 0"` 后机械臂是向后弯的，请改回 `rpy="0 -1.5708 0"`。

**建议：**
先在 Rviz2 中查看 `RobotModel`。如果实物在所有关节为 0 时是完全垂直的，那么 URDF 应该保持 `rpy="0 0 0"`。你只需在程序逻辑中记录：**“7”字型姿态的 Target Positions 是 `[0, 0, 1.57, 0, 0, 0]`**。
