### 🕵️ 真相揭秘：为什么这三个问题是连体婴儿？

1.  **为什么轨迹像“随便乱扭”？**
    你的机械臂底座是有高度的，当地面上的物体高度为 0（甚至紧贴着底座）时，这对于机械臂来说是一个极度别扭的“下腰”动作。
    MoveIt 默认使用的路径规划算法叫 **RRT（快速扩展随机树）**。当目标点在极限位置时，正常直来直去的路径会导致“胳膊肘撞到地面”或“关节超过最大旋转角度”。为了避开这些碰撞，算法只能在空间里随机乱拐，最终算出一条极其鬼畜、像麻花一样的轨迹。
2.  **为什么抓不准？**
    因为机械臂在极限姿态（奇异点附近）时，哪怕视觉算出的坐标只有 1 厘米的误差，机械臂为了够到那个点，关节也需要做出巨大的形变，导致末端偏移极其严重（误差被放大了）。
3.  **需要加个小桌子吗？**
    **太需要了！这是工业界绝对的标准化做法！** 你去任何一个无人工厂看，机械臂抓取的东西永远是在工作台或流水线传送带上的，绝对不可能让机械臂去地板上抠东西。把物体抬高到机械臂的“黄金操作区（胸口高度）”，轨迹自然就会变得无比丝滑。

### 🛠️ 终极改造：搭建工作台（只需三步）

我们现在就给你的仿真世界里搬一张桌子进去，让一切回归正轨！

#### 第一步：制作一张“桌子”的模型 (URDF)
在你的模型包里新建一个桌子的描述文件：
```bash
gedit ~/dummy_ws/src/dummy-ros2_description/urdf/table.urdf
```
把下面这段代码粘贴进去并保存（这是一个高 0.2 米、长宽 0.2 米的灰色方台）：
```xml
<?xml version="1.0"?>
<robot name="simple_table">
  <link name="table_base">
    <inertial>
      <mass value="50.0"/>
      <origin xyz="0 0 0"/>
      <inertia ixx="1.0" ixy="0" ixz="0" iyy="1.0" iyz="0" izz="1.0"/>
    </inertial>
    <visual>
      <origin xyz="0 0 0.1"/>
      <geometry>
        <box size="0.2 0.2 0.2"/>
      </geometry>
      <material name="grey">
        <color rgba="0.6 0.6 0.6 1"/>
      </material>
    </visual>
    <collision>
      <origin xyz="0 0 0.1"/>
      <geometry>
        <box size="0.2 0.2 0.2"/>
      </geometry>
    </collision>
  </link>
  <gazebo reference="table_base">
    <material>Gazebo/Grey</material>
  </gazebo>
</robot>
```

#### 第二步：在 Launch 文件中生成桌子和方块
打开 `~/dummy_ws/src/dummy-ros2_description/launch/gazebo.launch.py`。
找到我们之前生成小蓝块的地方（大概在第 35 到 45 行之间），将其**替换**为同时生成桌子和方块的代码：

```python
    # 1. 在正前方 0.4 米处生成桌子
    table_urdf_path = os.path.join(share_dir, 'urdf', 'table.urdf')
    spawn_table = Node(
        package='gazebo_ros', executable='spawn_entity.py',
        arguments=['-entity', 'simple_table', '-file', table_urdf_path, '-x', '0.40', '-y', '0.0', '-z', '0.0']
    )
    delayed_spawn_table = TimerAction(period=5.5, actions=[spawn_table])

    # 2. 将蓝块生成在桌子上方 (Z=0.25)，让它掉落在桌面上
    box_urdf_path = os.path.join(share_dir, 'urdf', 'target_box.urdf')
    spawn_box = Node(
        package='gazebo_ros', executable='spawn_entity.py',
        arguments=['-entity', 'target_box', '-file', box_urdf_path, '-x', '0.40', '-y', '0.0', '-z', '0.25']
    )
    delayed_spawn_box = TimerAction(period=6.0, actions=[spawn_box])
```
*别忘了在文件最后的 `return LaunchDescription([...])` 列表里，把 `delayed_spawn_table` 加进去。*

#### 第三步：调整 C++ 执行器的抓取高度
既然物体现在在 0.2 米高的桌面上，我们的悬停高度和抓取高度必须水涨船高。
打开 `~/dummy_ws/src/dummy_grasping/src/grasp_executor.cpp`，更新你的动作 2 和动作 3：

```cpp
  RCLCPP_INFO(logger, ">> 动作 2: 移动到方块上方 (安全悬停)...");
  // 悬停在方块上方（原先桌子高0.2 + 预留0.15安全距离 = 0.35）
  arm_group.setPositionTarget(target_x, target_y, target_z + 0.15); 
  arm_group.move();

  RCLCPP_INFO(logger, ">> 动作 3: 笔直下降...");
  // 下降到刚好夹住方块的高度
  arm_group.setPositionTarget(target_x, target_y, target_z + 0.02);
  arm_group.move();
```
