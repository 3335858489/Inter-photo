### 潜规则揭秘：为什么相机凭空消失了？
在 Gazebo 物理引擎中，如果一个模型部件（Link）**没有重量（缺少 `<inertial>` 标签）**，Gazebo 为了优化计算，会直接把它当成“虚拟垃圾”合并掉或者删掉！
你看看 `dummy-ros2.xacro` 里面的 `camera_link`，它只有 `<visual>`（外观），却没有 `<inertial>`（重量）。所以 Gazebo 在加载的时候，直接把你的相机给**“物理超度”**了，搭载在它上面的传感器自然发不出任何图像！

### 第一步：给相机加上重量（修改 URDF）
请打开 `dummy-ros2_description/urdf/dummy-ros2.xacro`，找到 `<link name="camera_link">`。
给它加上一段极小的 `<inertial>`。**请把原来的 `camera_link` 替换为：**

```xml
<link name="camera_link">
  <inertial>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <mass value="0.1"/>
    <inertia ixx="0.0001" iyy="0.0001" izz="0.0001" ixy="0.0" iyz="0.0" ixz="0.0"/>
  </inertial>
  <visual>
    <geometry>
      <box size="0.05 0.05 0.05"/>
    </geometry>
    <material name="silver"/>
  </visual>
</link>
```

### 第二步：修复 Launch 文件中的 3 个隐藏错误
你的 `gazebo.launch.py` 逻辑很棒，但有三个致命的小 Bug 导致你的控制器和抓夹不能正常工作：
1. **名字写错了：** 你的控制器名字写成了 `dummy_arm_controller`，但在 `ros2_controllers.yaml` 里，它叫 `arm_controller`。名字对不上，机械臂就动不了。
2. **抓夹被封印了：** 启动抓夹的代码 `# load_gripper_controller = Node(...)` 被你注释掉了。
3. **有两段 `return`：** Python 只要碰到第一个 `return` 就会结束，所以你最后面的代码是废代码。

**请清空你现在的 `gazebo.launch.py`，把这段完美修正版的代码粘贴进去：**

```python
import os
import xacro
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription, TimerAction
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch.substitutions import PathJoinSubstitution
from launch_ros.actions import Node
from launch_ros.substitutions import FindPackageShare

def generate_launch_description():
    share_dir = get_package_share_directory('dummy-ros2_description')

    xacro_file = os.path.join(share_dir, 'urdf', 'dummy-ros2.xacro')
    robot_description_config = xacro.process_file(xacro_file)
    robot_urdf = robot_description_config.toxml()

    robot_state_publisher_node = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        name='robot_state_publisher',
        parameters=[
            {'robot_description': robot_urdf,
             'use_sim_time': True}
        ]
    )

    gazebo = IncludeLaunchDescription(
        PythonLaunchDescriptionSource([
            PathJoinSubstitution([
                FindPackageShare('gazebo_ros'),
                'launch',
                'gazebo.launch.py'
            ])
        ]),
    )

    spawn_entity = Node(
        package='gazebo_ros',
        executable='spawn_entity.py',
        arguments=['-entity', 'dummy-ros2', '-topic', 'robot_description'],
        output='screen'
    )
    
    delayed_spawn = TimerAction(
        period=5.0,
        actions=[spawn_entity]
    )

    # 1. 启动状态广播器
    load_joint_state_broadcaster = Node(
        package="controller_manager",
        executable="spawner",
        arguments=["joint_state_broadcaster"],
        output="screen",
    )

    # 2. 启动机械臂控制器 (名字已修正为 arm_controller)
    load_arm_controller = Node(
        package="controller_manager",
        executable="spawner",
        arguments=["arm_controller"],
        output="screen",
    )

    # 3. 启动抓夹控制器 (解除封印！)
    load_gripper_controller = Node(
        package="controller_manager",
        executable="spawner",
        arguments=["gripper_controller"],
        output="screen",
    )

    # 依次延迟启动，确保 Gazebo 准备就绪
    return LaunchDescription([
        robot_state_publisher_node,
        gazebo,
        delayed_spawn,
        TimerAction(period=7.0, actions=[load_joint_state_broadcaster]),
        TimerAction(period=8.0, actions=[load_arm_controller]),
        TimerAction(period=8.5, actions=[load_gripper_controller]),
    ])
```
