### 第二步：复制核心算法代码

将下面的完整代码原封不动地粘贴到 `vision_grasp.py` 中并保存。

> **代码原理解析（向老师汇报时可以说）：**
> 这段代码利用了**针孔相机模型 (Pinhole Camera Model)** 的逆投影。我们先提取准星中心点的深度值 $Z$，然后利用相机的焦距 $(f_x, f_y)$ 和光心坐标 $(c_x, c_y)$，通过公式 $X = \frac{(u - c_x) \cdot Z}{f_x}$ 和 $Y = \frac{(v - c_y) \cdot Z}{f_y}$，将 2D 的像素坐标 $(u, v)$ 瞬间还原为相机坐标系下的 3D 物理坐标。

```python
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from cv_bridge import CvBridge
import cv2
import numpy as np

class VisionGraspNode(Node):
    def __init__(self):
        super().__init__('vision_grasp_node')
        self.bridge = CvBridge()
        self.latest_depth_image = None

        # 1. 订阅深度图话题
        self.depth_sub = self.create_subscription(
            Image,
            '/camera/rgbd_camera/depth/image_raw',
            self.depth_callback,
            10)

        # 2. 订阅彩色图话题
        self.color_sub = self.create_subscription(
            Image,
            '/camera/rgbd_camera/image_raw',
            self.color_callback,
            10)

        self.get_logger().info('🟢 视觉抓取节点已启动！请在弹出的窗口查看实时坐标...')

    def depth_callback(self, msg):
        # 将 ROS 的深度图转换为 OpenCV 可以计算的 32位浮点数矩阵 (单位: 米)
        try:
            self.latest_depth_image = self.bridge.imgmsg_to_cv2(msg, "32FC1")
        except Exception as e:
            self.get_logger().error(f'深度图转换错误: {e}')

    def color_callback(self, msg):
        # 必须等深度图准备好，才开始计算
        if self.latest_depth_image is None:
            return

        try:
            # 转换彩色图像格式为 BGR
            cv_image = self.bridge.imgmsg_to_cv2(msg, "bgr8")
            h, w, _ = cv_image.shape

            # 设定目标物体在画面的正中心 (你可以通过在 Gazebo 中移动方块让它对准这里)
            u, v = int(w/2), int(h/2)

            # 获取中心点的实际物理距离 (Z 轴距离)
            z = self.latest_depth_image[v, u]

            # 相机的内参 (根据你 xacro 里的 640x480 和 FOV=1.57 计算得出)
            fx, fy = 320.0, 320.0
            cx, cy = 320.0, 240.0

            # 如果深度值有效，进行 2D 像素到 3D 物理坐标的逆投影
            if not np.isnan(z) and z > 0.0:
                x = (u - cx) * z / fx
                y = (v - cy) * z / fy
                
                # 在画面左上角打印绿色的真实 3D 坐标
                text = f"Target 3D: (X:{x:.3f}m, Y:{y:.3f}m, Z:{z:.3f}m)"
                cv2.putText(cv_image, text, (30, 40), cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 255, 0), 2)
            else:
                cv2.putText(cv_image, "Scanning depth...", (30, 40), cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 0, 255), 2)

            # 在中心画一个红色准星
            cv2.drawMarker(cv_image, (u, v), (0, 0, 255), cv2.MARKER_CROSS, 20, 2)

            # 显示最终画面
            cv2.imshow("Dummy Robot - Vision AI", cv_image)
            cv2.waitKey(1)

        except Exception as e:
            self.get_logger().error(f'彩色图处理错误: {e}')

def main(args=None):
    rclpy.init(args=args)
    node = VisionGraspNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        cv2.destroyAllWindows()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 第三步：在终端中运行读取并测试

代码写好后，准备见证奇迹：

1. **终端 A（保持运行）：** 确保你的 Gazebo 仿真环境正在运行。
   ```bash
   ros2 launch dummy-ros2_description gazebo.launch.py
   ```
2. **移动方块：** 在 Gazebo 中，用平移工具把那个 0.04m 的小方块，尽量移动到画面的正中央位置。
3. **终端 B（运行 Python）：**
   ```bash
   # 切记每次新开终端都要 source 一下环境变量
   source /opt/ros/humble/setup.bash
   
   # 进入刚才建文件的目录
   cd ~/dummy_ws/src
   
   # 运行代码
   python3 vision_grasp.py
   ```

**你会看到什么？**
屏幕上会弹出一个 OpenCV 的视窗，中心有一个红色的十字准星。
当你把 Gazebo 里的小方块拖到准星下方时，左上角的绿色字体会立刻跳动，显示出这个方块距离相机的精确物理坐标（精确到毫米）！当你把方块拿走，只看到远处的地面时，坐标的 $Z$ 值会明显变大。
