你遇到的这个“摄像机反应不过来（画面卡死/掉帧）”的问题，可以说是虚拟机运行 Gazebo 时最让人头疼的顽疾。

当你在 Gazebo 的图形界面（GUI）里手动拖拽、缩放物体时，物理引擎会进行极其密集的实时碰撞检测和 UI 渲染。虚拟机的显卡驱动往往承受不住这种瞬间的计算爆发，导致整个仿真世界“假死”，相机话题也就跟着卡住了。

**专业的 ROS 开发者是如何解决这个问题的？**
答案是：**彻底抛弃手动拖拽！** 我们通过写代码的方式，让这个小方块作为一个独立的模型，在系统启动时**自动生成（Spawn）**在指定的坐标上。这样不仅完全绕过了 GUI 的渲染卡顿，还能保证每次做实验时，方块都在分毫不差的同一个位置！

按照下面简单的两步，我们把方块“焊死”在代码里：

### 第一步：为小方块创建一个专属模型文件

在你的模型文件夹里，我们专门为这个 4 厘米的小方块写一个 URDF 文件。

1. 打开终端，新建一个文件：
   ```bash
   gedit ~/dummy_ws/src/dummy-ros2_description/urdf/target_box.urdf
   ```
2. 把下面这段定义了“物理属性+4厘米外观+蓝色”的代码粘贴进去，保存并关闭：

```xml
<?xml version="1.0"?>
<robot name="target_box">
  <link name="box_link">
    <inertial>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <mass value="0.05"/>
      <inertia ixx="0.00001" iyy="0.00001" izz="0.00001" ixy="0" iyz="0" ixz="0"/>
    </inertial>
    <visual>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <box size="0.04 0.04 0.04"/>
      </geometry>
    </visual>
    <collision>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <box size="0.04 0.04 0.04"/>
      </geometry>
    </collision>
  </link>
  <gazebo reference="box_link">
    <material>Gazebo/Blue</material>
  </gazebo>
</robot>
```

### 第二步：修改 Launch 文件让它自动降落

现在我们需要告诉 Launch 启动脚本：“等机械臂加载完之后，顺便把刚才那个小方块也扔到桌面上”。

请打开你的 `~/dummy_ws/src/dummy-ros2_description/launch/gazebo.launch.py` 文件，**清空里面的内容，替换为下面这个加入了“自动生成方块”节点的完整版本：**

```python
import os
import xacro
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription, TimerAction
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch.substitutions import PathJoinSubstitution
from launch_ros.actions import Node
from launch_ros.substitutions import FindPackageShare

def generate_launch_description():
    share_dir = get_package_share_directory('dummy-ros2_description')

    xacro_file = os.path.join(share_dir, 'urdf', 'dummy-ros2.xacro')
    robot_description_config = xacro.process_file(xacro_file)
    robot_urdf = robot_description_config.toxml()

    robot_state_publisher_node = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        name='robot_state_publisher',
        parameters=[
            {'robot_description': robot_urdf,
             'use_sim_time': True}
        ]
    )

    gazebo = IncludeLaunchDescription(
        PythonLaunchDescriptionSource([
            PathJoinSubstitution([
                FindPackageShare('gazebo_ros'),
                'launch',
                'gazebo.launch.py'
            ])
        ]),
    )

    # 生成机械臂本体
    spawn_entity = Node(
        package='gazebo_ros',
        executable='spawn_entity.py',
        arguments=['-entity', 'dummy-ros2', '-topic', 'robot_description'],
        output='screen'
    )
    delayed_spawn = TimerAction(period=5.0, actions=[spawn_entity])

    # ================= 新增：自动生成目标小蓝块 =================
    box_urdf_path = os.path.join(share_dir, 'urdf', 'target_box.urdf')
    spawn_box = Node(
        package='gazebo_ros',
        executable='spawn_entity.py',
        arguments=[
            '-entity', 'target_box', 
            '-file', box_urdf_path, 
            '-x', '0.7',   # X 轴：放在机械臂正前方 0.7 米处
            '-y', '0.0',   # Y 轴：正中间
            '-z', '0.02'   # Z 轴：刚好贴在地面上
        ],
        output='screen'
    )
    delayed_spawn_box = TimerAction(period=6.0, actions=[spawn_box])
    # ==========================================================

    load_joint_state_broadcaster = Node(
        package="controller_manager",
        executable="spawner",
        arguments=["joint_state_broadcaster"],
        output="screen",
    )
    load_arm_controller = Node(
        package="controller_manager",
        executable="spawner",
        arguments=["arm_controller"],
        output="screen",
    )
    load_gripper_controller = Node(
        package="controller_manager",
        executable="spawner",
        arguments=["gripper_controller"],
        output="screen",
    )

    return LaunchDescription([
        robot_state_publisher_node,
        gazebo,
        delayed_spawn,
        delayed_spawn_box, # <--- 这里调用生成方块的指令
        TimerAction(period=7.0, actions=[load_joint_state_broadcaster]),
        TimerAction(period=8.0, actions=[load_arm_controller]),
        TimerAction(period=8.5, actions=[load_gripper_controller]),
    ])
```

### 🚀 重新起飞与测试

修改完这两个文件后，我们老规矩，重新编译并启动：

```bash
cd ~/dummy_ws
colcon build --symlink-install
source install/setup.bash
ros2 launch dummy-ros2_description gazebo.launch.py
```

**你会看到令人极度舒适的一幕：**
Gazebo 启动后约 5 秒，你的机械臂从天而降；紧接着第 6 秒，一个完美的 4 厘米蓝色小方块会自动“刷”在机械臂正前方 0.7 米的地上。

由于整个过程不需要你在 GUI 里做任何拖拽点击，虚拟机的性能压力被降到了最低，你的深度相机画面绝对会流畅如丝。此时你再另开一个终端运行刚才的 `python3 vision_grasp.py`，就能直接看到绿色的 $(X, Y, Z)$ 坐标实时在跳动了。
